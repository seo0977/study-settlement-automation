# study-settlement-automation

CPA 스터디그룹의 주간 공부시간 기록 및 벌금/상금 정산을 자동화하는 프로젝트입니다.

## Background

스터디그룹을 운영하며 매주 아래 과정을 엑셀로 수작업 처리했습니다.

- 스터디원별 요일당 공부시간 기록
- 주간 합계 계산 후 기준시간(주 55시간) 대비 초과/미달 판정
- 미달 시 벌금, 초과 시 상금 산정
- 전주 이월 벌금/상금을 다음 주 정산에 반영
- 매주 새 시트를 복사해 위 과정을 반복

48주 이상 이 과정을 반복하며 매주 동일한 계산을 수기로 처리하는 데 상당한 시간이 소요됨을
경험했고, 규칙이 명확한 반복 작업이라는 점에서 자동화가 가능하다고 판단했습니다.

## Goal

- 주간 공부시간 데이터를 입력하면 벌금/상금/이월 금액을 자동 산출
- 기존 엑셀 로직을 Python으로 재구현
- (추후) 자연어로 정산 내역을 질의할 수 있는 기능 추가 예정

## How it works

기존 방식(엑셀)에서는 매주 시트를 복사해 개인별 공부시간을 기록하고,
기준 미달 시 벌금을 수기로 계산했습니다. 벌금은 즉시 걷어 나누지 않고
개인별로 누적시켰다가, 학기 말에 한 번에 정산하는 방식으로 운영되었습니다.

`settlement.py`는 이 과정을 그대로 코드로 옮겼습니다.

### 1. 주간 처리 (`process_week`)

한 주치 데이터(스터디원별 공부시간, 상벌점)를 입력하면:

- 기준 시간(예: 55시간, 시험 임박 시 62시간으로 상향) 대비 부족한 시간을 계산
- 부족한 시간 1시간당 1,000원의 벌금을 개인 잔액에 가산
- **상금 자격**: 기준 시간 이상 달성 + 상점·벌점 합계가 0 이상인 경우 인정
- 이번 주 변동분이 개인 잔액에 누적됨 → **기초잔액 + 이번주 변동 = 기말잔액**

### 2. 최종 정산 (`final_settlement`)

학기 동안 쌓인 벌금 pot을 각자의 **상금 자격 누적 주차 수** 비례로 분배하고,
개인별 최종 순잔액(납부할 금액 또는 수령할 금액)을 계산합니다.

## Why this matters

매주 반복되는 계산을 수기로 처리하면서 실수 가능성과 시간 소모가 컸습니다.
동일한 규칙을 코드로 재현하면서, 정산 기준(기준시간·벌금 단가·이월 방식)을
바꿀 때도 코드 수정만으로 대응할 수 있도록 구조화했습니다.

## Example Output

`examples/real_data_sample.py`를 실행하면 3주치 데이터를 기반으로 아래와 같은 최종 정산 결과가 출력됩니다.

\`\`\`
===== 최종 정산 (누적 pot: 75,000원) =====

- Member3: 벌금누적 31,000원 / 상금자격 0주 / 분배 0원 → 31,000원 납부
- Member9: 벌금누적 17,000원 / 상금자격 1주 / 분배 4,688원 → 12,312원 납부
- Member1: 벌금누적 0원 / 상금자격 3주 / 분배 14,062원 → 14,062원 수령
- Member5: 벌금누적 0원 / 상금자격 3주 / 분배 14,062원 → 14,062원 수령
- Member8: 벌금누적 0원 / 상금자격 3주 / 분배 14,062원 → 14,062원 수령
\`\`\`

3주 내내 상금 자격을 유지한 멤버(Member1, 5, 8)가 가장 많은 금액을 수령하고,
지속적으로 기준 미달이었던 멤버(Member3)가 가장 많은 금액을 납부하는 것을
확인할 수 있습니다.
